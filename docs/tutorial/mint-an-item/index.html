Mint an existing item

Note: to follow this tutorial, simply copy/paste the curl requests, or use the Postman collection located here

We have now created an item in the item store and we want to move it to the blockchain. Our item has an id which is a number greater than 1. It was asigned when the item was first created. Moving the item to the blockchain will effectively create our first NFT!

% curl --location --request PUT 'http://localhost:8000/api/tokenization-service/warehouse/<item id>'

Which will return the object as it was retrieved from the item store:

{
"available_quantity": 1,
"data": {
"level": "legendary",
"type": "goose",
"attribute": "gold"
},
"item_id": <item id>,
"name": "Golden Goose",
"total_quantity": 1
}

This requests takes some time to execute, because writing an item in the blockchain is slow and the request is synchronous, it will only complete when the operation completes in the Tezos blockchain. Also note the PUT request and the item_id in the URL. There is no need to describe the item, its current state will be read directly from the item store.

We have now tokenized, or minted, our first item as an NFT. Let take a look in the blockchain!

Get Warehouse smart contract address#

curl --location --request GET 'http://localhost:8000/api/tokenization-service/info'

which returns:

{

}

Get a warehouse item by id#

We know the id of our warehouse item, instead of looking directly inside the blockchain, let's retrieve it via a rest call:

curl --location --request GET 'http://localhost:8000/api/tokenization-service/warehouse/1'

Which returns

{
"available_quantity": 1,
"name": "Golden Goose",
"item_id": 1,
"total_quantity": 1,
"data": {
"attribute": "gold",
"level": "legendary",
"type": "goose"
}
}
